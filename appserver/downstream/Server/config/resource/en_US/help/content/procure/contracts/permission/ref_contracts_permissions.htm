Contracts permissions
This permission Authorizes users to
ContractManager

View and edit all contract requests (CRs), view, open, and close all contracts, and receive against contracts. By default, this permission is assigned to these roles: Contract Manager, Professional Buyer, and System Administrator.

Users assigned the Contract Manager role also have permission to create sourcing requests.

Only users who have the CreateContract or ContractManager permission can be given edit access to a contract. The creator of a contract is automatically given edit access to the contract.

ContractTypeImpactReport Receive an impact report when a catalog type is used by a contract and a new version of the catalog type is uploaded that might affect existing contracts that use that type.
CreateContract

Create a Contract Request (CR) and, if the user is the requester or preparer, or is listed on the CR Edit Access list, change a CR or open and close a contract. This permission applies only in configurations using Ariba Contract Compliance. By default, this permission is assigned to these roles: Contract Agent, Contract Manager, and Category Manager.

Users with this permission receive notification messages when a contract they are associated with is opened, closed, nearing its expiration date, or when it reaches a specified percentage of the original amount.

Only users who have the CreateContract or ContractManager permission can be given edit access to a contract. The creator of a contract is automatically given edit access to the contract.
DirectReleaseContract

View the Contracts tab when adding items to a requisition, and create releases against a contract selected on the tab. This permission applies only in configurations using Ariba Contract Compliance. By default, this permission is assigned to these roles: Contract Agent, Contract Manager, and Category Manager.

SourcingAuthorized

To create a spot purchase sourcing request (SR) when they are asked to approve a purchase request that contains non-catalog items, and create contract sourcing requests (CSRs). By default, this permission is assigned to these roles: Professional Buyer and Contract Manager.

Authorized users also receive email notification messages from Ariba Contract Compliance and from Ariba Enterprise Sourcing as sourcing requests they have created proceed through the approval and sourcing event processes.

SourcingPricingTermErrorsEmail Receive the Error in Pricing Terms notification message when Ariba Contract Compliance encounters errors related to pricing terms. The body of the notification message describes the type of error. By default, this permission is assigned to the role Professional Buyer.
