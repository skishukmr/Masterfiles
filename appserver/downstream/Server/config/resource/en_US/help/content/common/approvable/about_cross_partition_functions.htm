Tips   About actions that cross partitions

If your company uses multiple Enterprise Resource Management (ERP) systems, My Supply Cabinet might be partitioned into different sections that correspond to each ERP system. In this case, you are prompted to select a partition in which to work after you log in.

Expiring contracts
 

Typically, data created in one partition can be accessed only when you are logged into that partition. In other words, with few exceptions, you cannot access data across partitions. The exceptions to this rule are:

  • If you are an approver, you can approve requests from any partition. However, to edit a request while you approve or deny it, you must be logged in to the same partition in which the request was created.
  • If you need to run reports that contain data from multiple partitions, confirm with your System Administrator that the report metadata indicates "IsCrossPartition." This allows you to run one report for multiple partitions, instead of logging in to separate partitions to run the same report multiple times.
  • If you can view expiring contracts from your dashboard, and have access to multiple partitions, you can change partitions using the partition pull-down menu.
