Procurement permissions
This permission Authorizes users to
AdvancedShipNoticeErrorNotification

Receive an email notification message (Notification 75) when a ship notice message is in a different partition than the related purchase order. This permission applies only for configurations that retrieve ship notice messages from Ariba Supplier Network (Ariba SN). By default, this permission is assigned to the Professional Buyer, Receiving Manager, and System Administrator roles.
AdvancedShipNoticeSuccessNotification

Receive an email notification message (Notification 78) when a ship notice message is successfully received for a purchase order. This permission applies only for configurations that retrieve ship notice messages from Ariba SN. By default, this permission is assigned to the Professional Buyer and Receiving Manager roles.
CancelPO

Cancel a purchase order generated from a requisition you submitted. If you are a purchasing agent or an administrative user, use this permission in conjunction with the RequisitionChange permission to cancel an order generated from a requisition submitted by another user.

ChangePO

Change a purchase order generated from a requisition you submitted. If you are a purchasing agent or an administrative user, this permission in conjunction with the RequisitionChange permission enables you to change an order generated from a requisition submitted by another user.

CreateRequisition

Create purchase requisitions. This permission is not used in the default configuration. If the My Supply Cabinet PunchIn site is enabled, this permission can be used to prevent supplier users from creating requisitions.
ForceProcess Force a purchase order to the Ordered status when there is a problem in the ordering process, or to the Canceled status when there is an interruption in the normal process.
NoNonCatalogItems Order catalog items only; users with this permission cannot order non-catalog items. Note that this permission does not control whether a user is able to create non-catalog items on a contract.
OrderConfirmationErrorNotification

Receive an email notification message (Notification 74) when the unit of measure in the order confirmation message is different than the unit of measure in the purchase order. This permission applies only in configurations that retrieve order confirmation messages from Ariba SN. In the default configuration, this permission is assigned to the roles Professional Buyer and System Administrator.

OrderConfirmationSuccessNotification Receive (Notification 77) when an order confirmation message is successfully received for a purchase order. This permission applies only in configurations that retrieve order confirmation messages from Ariba SN. In the default configuration, this permission is assigned to the the Professional Buyer role.
OrderFailureImmediateEmail

Receive email notification messages (Notification 20 and Notification 23) when an order being sent to a supplier experiences transmission problems. Users also receive Notification 50 whenever an order goes to Ariba SN with an attachment that is not acceptable to Ariba SN. This permission is typically assigned to purchasing agents.

PurchaseOrderCancelEmail

Receive an email notification message (Notification 35) when a user deletes all of the line items from an order (essentially canceling the order) that was pushed to an ERP system and canceling orders is not enabled for the ERP. This permission is typically assigned to purchasing agents, so the purchasing agent can undo the deletion of at least one of the line items, ensuring the order is not canceled and a change order is sent.

PurchaseOrderChangeEmail Receive an email notification message (Notification 58) when change order integration with ERPs is disabled after a user has created a change order but before it has been sent to the ERP. This permission is typically assigned to purchasing agents, so the purchasing agent can cancel the change order.
PurchaseOrderRejectionEmail

Receive an email notification message (Notification 34) whenever users reject items from a purchase requisition. This permission is typically assigned to purchasing agents.

ReceiveError Receive Notification 24 when a receipt was not successfully pushed to an ERP. When such an error occurs, the receipt remains in a “processing” state and My Supply Cabinet sends a notification message to the specified permission. In the default configuration, users with the ReceiveError permission also have the QueryAll permission so receipt managers can search for and edit any receipt in the system, (including auto-receipts).
RequisitionChange

Change a requisition that was submitted by another user, and that has been fully approved. By default, the requester and preparer of a requisition are able to change a requisition. This permission authorizes additional administrative users, such as purchasing agents, to be able to create change orders on other users’ requisitions.

Changing a requisition changes the related purchase orders as well. To cancel or change a specific purchase order generated from a requisition, the user must also have the CancelPO or ChangePO permission.

SupplierLocationCheckEmail Receive an email notification message (Notification 16) when incorrect supplier contact information, such as an invalid fax number or missing phone number, exists. This permission is typically assigned to purchasing agents.
