Tips   Notification 21: Order Rejection

If the request contains line items split into several orders, only the items from the order that was rejected are editable. If the other orders successfully reached the ERP, the preparer can only make changes to them by creating a change order.

About notification messages

Notification messages
 

When an ERP system rejects an order sent by My Supply Cabinet, My Supply Cabinet emails a notification message to the preparer of the purchase request that the order has been rejected. Users who have the permission specified by the parameter Application.Procure.ERPPushFailNotificationPermission in the config/Parameters.table file also receive the notification.

The message identifies the request that caused the error, the time the error occurred, the cause of the error, and the action required to resolve the error.

In some cases, the status of the request reverts to Composing. The preparer (or an administrator) must correct the error and resubmit the requisition.
