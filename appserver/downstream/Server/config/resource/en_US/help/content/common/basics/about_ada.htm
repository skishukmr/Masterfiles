Tips   About accessibility features

Accessibility features are available only if your company has enabled them and you are assigned the AccessibilityEnabled permission.

Learn more about Ariba's efforts to make its products accessible to disabled users at www.ariba.com/accessibility/index.htm.

About permissions
 

In an effort to make content more accessible to users with disabilities and more compatible with assistive technology, such as screen readers, Ariba typically provides these accessibility features in its products:

  • Navigational and informational images (such as page controls or icons) provided by Ariba use alt or title attributes to display a descriptive string when you move the cursor over them or navigate to them using the Tab key. Ariba uses alt and title attributes on many page elements.
  • The label tag is used to explicitly associate label strings with their respective input fields.
  • Row and column headings in data tables provided by Ariba use <th> tags and the scope attribute to associate row and column data with the corresponding heading.
  • When color is used to communicate information, as in a red error message, other methods are also used. For example, error messages formatted using a bold red font are also preceded by an alt-tagged warning icon or use the term "error" in the message text.
  • When framesets are used, the frames are titled to facilitate their identification and navigation.
  • You can use the keyboard to tab through the product interfaces and address or select the control or element that has the current focus.
